Cortex-M4 Exception Handlers: Step-by-Step Vetting Guide

01. Thread Mode (Idle)

The processor runs user/application tasks in **Thread Mode** under low privilege levels. The active Stack Pointer points to **MSP** base (`0x20008000`), and IPSR reads `0` (indicating no exceptions are active).

02. Interrupt Assertion

An external peripheral asserts an interrupt line, writing a `1` to the pending register bit (`NVIC_ISPR0`). If enabled in `NVIC_ISER0`, the NVIC compares the priority in `NVIC_IPR` and triggers a CPU preemption if it exceeds the current execution level.

03. Context Stacking

Before branching to the vector address, CPU hardware **automatically pushes an 8-word stack frame** onto the active MSP stack: `xPSR`, `PC (Return Addr)`, `LR`, `R12`, `R3`, `R2`, `R1`, `R0`. The Stack Pointer (SP) automatically decrements by 32 bytes (to `0x20007FE0`).

04. Vector Entry (ISR)

The CPU fetches the starting instruction of the active Interrupt Service Routine (ISR) from the Vector Table (`0x080009A0 + vector_idx * 8`) and loads it into the `PC`. The Link Register (`LR`) is written with the exception return hook `EXC_RETURN` (`0xFFFFFFF9`). IPSR transitions to active exception number.

05. Exception Return

Upon ISR completion, the instruction `BX LR` executes. Because `LR` holds `0xFFFFFFF9`, the hardware unstacking routine triggers: the CPU automatically pops the 8-register context stack frame back from SRAM, increments `SP` by 32 bytes to `0x20008000`, and resumes thread mode.
